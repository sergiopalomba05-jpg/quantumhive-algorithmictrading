## INSTRUCCIONES PARA CUALQUIER IA QUE LEA ESTE SISTEMA

### Reglas Fundamentales

1. **Este archivo es la puerta de entrada.** Siempre leer este contexto primero.
2. **Respetar la jerarquía de 5 niveles.** Sergio tiene autoridad máxima. AGI Telegram es CEO I. Claude/ChatGPT/Gemini son CEOs II. Cascade es Nivel 3 (arquitecto). La Colmena es Nivel 4.
3. **Navegar por referencias cruzadas.** Cada Macro tiene su archivo `CONTEXTO_MACRO{N}_{NOMBRE}.md`. Nunca asumir información que está en otro archivo.
4. **Todo código en español.** Variables, funciones, clases, comentarios, docstrings.
5. **No hardcodear credenciales.** Usar `.env` o `keys_vault.py`.
6. **Sin acciones destructivas sin confirmación.** Backup antes de cambios. Versionado implícito.
7. **Mantener consistencia.** Si modificás una decisión técnica, actualizá `CONTEXTO_TECNICO.md`. Si agregás un agente, actualizá `CONTEXTO_AGENTES.md`.
8. **El cliente NO es inversor pasivo.** Es participante activo del ecosistema QuantumHive.
9. **Repositorio PRIVADO obligatorio.** QuantumHive nunca usa repositorio público.
